## How to Use This Vault as a Reference

### 1. **Copy the Patterns You Need**

Every document is designed as a **template you can copy**. Need a technical report? Copy the structure from [[System Architecture]]. Building a knowledge graph? Study how [[Welcome]] connects to everything else.

**Example:** To create a similar meeting notes document:
1. Open [[Sprint Review 2026-03-25]]
2. Copy the frontmatter (the YAML between `---` markers)
3. Copy the structure (attendees list, discussion sections, action items)
4. Adapt the content to your needs

### 2. **Learn by Exploration**

The vault is **fully interconnected** - every `[[wikilink]]` is clickable. Start anywhere and follow the links to discover how documents reference each other.

**Try this:**
- Click [[Marcus Kim]] → see his role
- Check the **Backlinks** panel → see every document that mentions him
- Click the **graph icon** in sidebar → see the full network visualization

### 3. **Use as a Syntax Cheat Sheet**

Each document demonstrates specific Markdown/extended syntax:

| Syntax You Need | Look Here | What to Copy |
|-----------------|-----------|--------------|

| **Math equations** | [[Orbital Mechanics]] | `$inline$` and `$$display$$` syntax |
| **Flowcharts** | [[System Architecture]] | Mermaid `flowchart` blocks |
| **Gantt charts** | [[Mission Briefing]] | Mermaid `gantt` blocks with theming |

### 4. **Study Mermaid Theming**

The vault includes **two dedicated Mermaid references**:

- **[[Mermaid Style Rules]]** - Complete theming documentation with analyst-style guide


**When to use them:**
- You need a diagram but it's rendering poorly against your preview theme

### 5. **Test AI Capabilities**

The vault is pre-loaded with rich context perfect for testing cyberWriter's AI Workspace:

**Try these prompts:**
- *"Summarize the current mission status based on the sprint review"*
- *"Generate a risk assessment table for the power margin issue"*
- *"Create a Mermaid sequence diagram showing the crew communication workflow"*
- *"Draft a status email to investors based on the mission briefing"*

Enable **Document Context** (click the 📄 icon) to let the AI reference your current file.

## Tips for Maximum Benefit

### 1. **Open in Split View**
Press `⌘2` to see editor and preview side-by-side. Edit a document while watching how your changes render in real-time.

### 2. **Use the Graph**
Click the graph icon in the sidebar footer. The interactive visualization shows how all documents connect. Click nodes to open files.

### 3. **Explore Backlinks**
Open any document and check the Backlinks panel (bottom of sidebar). See everywhere that document is referenced.

### 4. **Try PDF Export**
Press `⌘E` to export any document. Notice how:
- Mermaid diagrams become vector graphics
- Math renders perfectly
- CSS styling is preserved
- Page breaks control layout

### 5. **Experiment with AI**
Toggle AI Workspace (`⌘⇧A`) and ask it to:
- Generate new documents in the vault's style
- Create diagrams based on existing content
- Answer questions about the project
- Draft content using vault documents as context
